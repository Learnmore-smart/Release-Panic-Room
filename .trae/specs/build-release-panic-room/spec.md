# 周五上线修罗场 (Release Panic Room) Spec

## Why
在真实的互联网开发环境中，“周五傍晚准备上线”往往伴随着高压和不可预见的突发状况。本项目旨在通过一个互动 Web 游戏，模拟这种高压场景下负责人的连续决策过程。这不仅是一个具有职场共鸣的娱乐产品，更是一个具备产品感、有压迫感且极具传播性的决策模拟器，让用户在极短时间内体验上线前的生死时速，并获得个性化的分享卡片。

## What Changes
- **全新构建**：使用 Next.js (App Router) + TypeScript + Tailwind CSS 构建独立 Web 应用。
- **游戏引擎设计**：在纯前端实现基于状态机的决策系统，不依赖后端，包含本地数据结构定义（事件、选项、效果、结局）。
- **状态与时间系统**：引入 5 个核心状态指标（发布信心、风险值、团队信任、用户影响、混乱值）和时间推进机制（从周五 17:42 开始）。
- **事件内容库**：设计 8-10 个真实感极强的突发事件（如高优 bug、监控 warning、核心同事离线等），每个事件 3 个选项，附带即时反馈。
- **结局生成与分享卡**：设计 8-10 种具有传播属性的结局（如“回滚先知”、“周五纵火犯”等），并在结果页生成极具设计感、适合截图分享的总结卡片。
- **UI/UX 体验**：采用深色系“控制室 / 战情室”视觉风格，通过排版、色彩（深灰/墨黑为主，红绿点缀）和微动效营造高级、克制且紧张的氛围。

## Impact
- Affected specs: 全新项目，涉及前端页面渲染、状态管理、内容配置。
- Affected code:
  - `/app` 目录下的路由页面 (Home, Game, Result)
  - `/components` 目录下的 UI 组件 (Dashboard, EventCard, ShareCard)
  - `/lib` 或 `/data` 目录下的游戏状态逻辑与配置数据

## ADDED Requirements
### Requirement: 核心游戏流程
系统 SHALL 提供一个流畅且紧张的 60-90 秒决策体验，包括首页引入、连续事件决策和结局展示。

#### Scenario: 玩家完成一局游戏
- **WHEN** 用户点击“开始这一局”
- **THEN** 进入游戏主界面，时间设定为 17:42，状态值初始化。
- **WHEN** 用户对当前事件做出选择
- **THEN** 系统更新状态值，推进时间，显示简短反馈，并在短暂延迟后展示下一个事件。
- **WHEN** 用户完成所有事件决策
- **THEN** 系统根据最终状态值计算结局，并展示包含称号、总结和分享卡的结果页面。

### Requirement: 数据与状态管理
系统 SHALL 在前端完全维护游戏状态，包括各指标数值、当前时间、事件历史和决策记录。

### Requirement: 视觉与传播
系统 SHALL 提供高质量的暗色系 UI 设计，结果页面必须包含易于截图和分享的卡片组件。
